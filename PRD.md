**產品名稱** Antigravity Optical ID Reader
**版本** v1.0
**目標** 在免安裝（Web）條件下，透過手機相機讀取 LED 光學編碼，穩定解析 96-bit 資料並取得 16-bit 核心 ID。

---

**1. 背景與問題定義**

LightTag 結構：
- 3 個數據燈條（用於 96-bit 光學編碼）
- 5 個藍燈定位燈珠（4 顆平面 + 1 顆中央突起）

資料廣播：
- 3 顆 LED 以時間序列方式廣播資料
- 完整資料長度 96 bit，分佈於 32 個 frame（每 frame 3 bit）

限制與需求：
- 發光採用人眼不可見調變，需依賴相機特定條件取樣
- 使用者不可安裝 App
- 必須透過瀏覽器完成讀取與解析
- 讀取時間需在可接受範圍（< 3 秒）
- 成功率需可量產（> 95% in supported devices）

---

**2. 產品目標（Goals）**

必達：
- 成功取得 16-bit ID
- Web 端完成全流程
- 自動完成幀同步與校驗
- 對使用者無需理解曝光/相機術語

非目標（v1 不處理）：
- 任意手機全面兼容
- 在所有 AE / HDR 開啟情況下保證成功
- 背景強光極端場景

---

**3. 使用流程（User Journey）**

1. 使用者打開網頁
2. 允許相機權限
3. 系統自動檢測設備能力
4. 若支持 → 進入讀取
5. 若不支持 → 給出明確提示
6. 1~3 秒內返回 ID
7. 進行後續業務流程

---

**4. 系統總覽（High Level）**

LED 發送端 → 光學 → 相機 Sensor → Browser MediaStream → Frame Grabber → 藍燈定位偵測（5 點星座） → 6DoF / PnP 解算（定位） → LED ROI Extractor → Threshold / Symbol Decode → Preamble Sync → Packet Builder（32 frames） → CRC / ID Validate → Output ID

---

**5. 功能需求**

**5.1 相機能力管理（最關鍵）**

FR-1：能力探測
- frameRate range
- exposureMode / exposureTime
- iso / gain
- focusMode

FR-2：條件判定
- fps ≈ 25
- exposure 可接近短曝光（~1/50）
- 可穩定輸出連續影像

FR-3：參數嘗試鎖定
- frameRate = 25
- exposure ≈ 1/49
- ISO ≈ 750
- 並讀回 settings 驗證

**5.2 LED 偵測**

FR-4：ROI 初始化
- 自動或半自動定位三顆 LED

FR-5：亮度提取
- 每 frame 需取得三個 ROI 的亮度值

**5.3 Symbol 解碼**

FR-6：自適應門檻
- 每顆 LED 需獨立 threshold，避免環境光與 AE 漂移

FR-7：生成 symbol
- 每 frame 產生 3bit → 0~7

**5.4 幀同步**

FR-8：Preamble 搜尋
- 使用滑動窗口從連續 frame 中定位封包起點

FR-9：允許丟幀
- 系統需能在短暫錯位後重新同步

**5.5 封包與驗證**

FR-10：收集 32 frames
FR-11：生成 96 bit
FR-12：CRC 驗證
FR-13：抽取 16-bit ID
FR-14：若 CRC fail → 丟棄並等待下一輪

**5.6 輸出**

FR-15：在成功後 100ms 內輸出 ID

**5.7 光定位（5 藍燈）**

FR-16：藍燈幾何星座
- 基座 4 顆：共面矩形分佈（參考平面）
- 中央 1 顆：向前突起、非共面（提供視差）

FR-17：定位演算法
- 使用 5 點 2D 影像座標與已知 3D 座標，進行 PnP 解算取得 6DoF 位姿

FR-18：定位前置條件
- 未偵測到 5 點幾何星座前，不啟動數據解碼流程（先定位、後解碼）

FR-19：藍燈偵測策略
- HSV 色相區間（約 210°–250°）
- 藍通道差分：Blue - (Red + Green)/2

FR-20：定位精度目標
- 在 2–4 公尺範圍內，目標 ≤ 10 cm 定位誤差（使用亞像素中心提取）

**5.8 Web 定位實作補充（v1 現況）**

FR-21：影像前處理（定位專用）
- 藍燈遮罩視圖（僅保留藍色特徵點）
- HSV 手動調整（Hue / Sat / Val）
- 自動 HSV 校準（可關閉）

FR-22：候選點偵測策略（長距離）
- 主偵測使用藍色差分：Blue - (Red + Green)/2
- 自適應閾值（二值化）+ 形態學去噪
- 圓度、面積、中心優先等條件過濾

FR-23：幾何導向匹配
- 從候選點枚舉 5 點組合
- 用 PnP 重投影誤差評分，選最小誤差組合
- 若無合格組合，回退重新偵測

FR-24：持續偵測與追蹤
- 自動偵測定位燈為持續狀態（非單次）
- 偵測成功後進入追蹤（小窗內更新）
- 偵測節流以避免畫面卡頓

FR-25：輸出資訊
- 輸出 6DoF 位姿（PnP）
- 顯示距離（m）= sqrt(x^2 + y^2 + z^2) / 1000
- 顯示解析度與藍燈品質（像素直徑 / SNR）

FR-26：解析度控制
- 提供 1080p / 1440p / 4K 解析度選擇
- 實際輸出以裝置回傳為準

**7. 相機約束（現實）**

iOS：
- 幾乎不可鎖 exposure / ISO
- 需依賴 decoder 容錯

Android：
- 部分機型可透過 constraints 接近鎖定
- 必須逐機型驗證

---

**8. 風險（必須讓管理層看到）**

R1：Web 對底層 ISP 控制不足，成功率會隨手機差異波動
R2：HDR / multi-frame NR 可能抹平調變
R3：fps 可能被 thermal policy 改變
R4：rolling shutter 行為各家不同

---

**9. 風險緩解策略**

- 重複廣播
- 強 preamble
- ID 多次一致性
- 快速重試
- 設備白名單
